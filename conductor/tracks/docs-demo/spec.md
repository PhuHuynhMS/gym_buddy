# Docs Demo Spec

## Goal

Make the project understandable, reproducible, and demo-ready.

## Technical Requirements

- Documentation must not contain real secrets.
- Setup steps should cover both `gym_buddy_server` and `gym_buddy_app`.
- Diagrams should match the implemented architecture.
- Demo checklist should map to finished Conductor tasks.

## Blockers

- Final API surface for Swagger or equivalent documentation.
- Deployed or recorded demo target environment.
